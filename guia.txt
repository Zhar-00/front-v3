## 2. Autenticación

### 🔐 2.1 Login

**Endpoint:** `POST /api/auth/login`

**Usuario:** Público (sin token)

**Request:**
```json
{
  "email": "admin@sigesto.com",
  "password": "123456"
}
```

**Response (200 OK):**
```json
{
  "mensaje": "Inicio de sesión exitoso.",
  "token": "1|abc123xyz...",
  "usuario": {
    "id": 1,
    "nombre_completo": "Carlos Administrador",
    "email": "admin@sigesto.com",
    "id_rol": 1,
    "nombre_rol": "ADMINISTRADOR"
  }
}
```

**Response (422 - Error):**
```json
{
  "mensaje": "Error de autenticación",
  "errores": {
    "email": ["Las credenciales proporcionadas son incorrectas."]
  }
}
```

#### 🟢 Java (Android):
```java
// Agregar en build.gradle
implementation 'com.squareup.okhttp3:okhttp:4.12.0'
implementation 'com.google.code.gson:gson:2.10.1'

public class ApiClient {
    private static final String BASE_URL = "http://10.0.2.2:8000/api";
    private static final OkHttpClient client = new OkHttpClient();
    private static final Gson gson = new Gson();
    private static String authToken = null;
    
    public static void setToken(String token) {
        authToken = token;
    }
    
    public static void clearToken() {
        authToken = null;
    }
    
    public static String postJson(String endpoint, Object body) throws IOException {
        String json = gson.toJson(body);
        RequestBody requestBody = RequestBody.create(
            json, MediaType.parse("application/json")
        );
        
        Request.Builder builder = new Request.Builder()
            .url(BASE_URL + endpoint)
            .addHeader("Accept", "application/json")
            .post(requestBody);
        
        if (authToken != null) {
            builder.addHeader("Authorization", "Bearer " + authToken);
        }
        
        Request request = builder.build();
        try (Response response = client.newCall(request).execute()) {
            return response.body().string();
        }
    }
}

// Uso:
Map<String, String> credentials = new HashMap<>();
credentials.put("email", "admin@sigesto.com");
credentials.put("password", "123456");

try {
    String response = ApiClient.postJson("/auth/login", credentials);
    JsonObject json = JsonParser.parseString(response).getAsJsonObject();
    
    String token = json.get("token").getAsString();
    ApiClient.setToken(token);
    
    System.out.println("Login exitoso: " + token);
} catch (IOException e) {
    e.printStackTrace();
}
```

#### 🔵 Flutter (Dart):
```dart
// pubspec.yaml
dependencies:
  http: ^1.2.0

import 'package:http/http.dart' as http;
import 'dart:convert';

class ApiClient {
  static const String baseUrl = 'http://10.0.2.2:8000/api';
  static String? _authToken;
  
  static void setToken(String token) {
    _authToken = token;
  }
  
  static void clearToken() {
    _authToken = null;
  }
  
  static Map<String, String> get _headers => {
    'Content-Type': 'application/json',
    'Accept': 'application/json',
    if (_authToken != null) 'Authorization': 'Bearer $_authToken',
  };
  
  static Future<Map<String, dynamic>> post(String endpoint, Map<String, dynamic> body) async {
    final response = await http.post(
      Uri.parse('$baseUrl$endpoint'),
      headers: _headers,
      body: jsonEncode(body),
    );
    
    final data = jsonDecode(response.body) as Map<String, dynamic>;
    
    if (response.statusCode >= 200 && response.statusCode < 300) {
      return data;
    } else {
      throw ApiException(
        message: data['mensaje'] ?? 'Error desconocido',
        statusCode: response.statusCode,
        errors: data['errores'],
      );
    }
  }
}

class ApiException implements Exception {
  final String message;
  final int statusCode;
  final dynamic errors;
  
  ApiException({required this.message, required this.statusCode, this.errors});
  
  @override
  String toString() => 'ApiException($statusCode): $message';
}

// Uso:
try {
  final response = await ApiClient.post('/auth/login', {
    'email': 'admin@sigesto.com',
    'password': '123456',
  });
  
  final token = response['token'] as String;
  ApiClient.setToken(token);
  
  print('Login exitoso: $token');
} on ApiException catch (e) {
  print('Error ${e.statusCode}: ${e.message}');
}
```

#### 🟡 JavaScript (Fetch API):
```javascript
class ApiClient {
  constructor(baseUrl = 'http://localhost:8000/api') {
    this.baseUrl = baseUrl;
    this.authToken = localStorage.getItem('authToken');
  }
  
  setToken(token) {
    this.authToken = token;
    localStorage.setItem('authToken', token);
  }
  
  clearToken() {
    this.authToken = null;
    localStorage.removeItem('authToken');
  }
  
  get headers() {
    const headers = {
      'Content-Type': 'application/json',
      'Accept': 'application/json',
    };
    if (this.authToken) {
      headers['Authorization'] = `Bearer ${this.authToken}`;
    }
    return headers;
  }
  
  async post(endpoint, body) {
    const response = await fetch(`${this.baseUrl}${endpoint}`, {
      method: 'POST',
      headers: this.headers,
      body: JSON.stringify(body),
    });
    
    const data = await response.json();
    
    if (!response.ok) {
      throw new ApiError(data.mensaje || 'Error', response.status, data.errores);
    }
    
    return data;
  }
}

class ApiError extends Error {
  constructor(message, statusCode, errors) {
    super(message);
    this.statusCode = statusCode;
    this.errors = errors;
  }
}

// Instancia global
const api = new ApiClient();

// Uso:
try {
  const response = await api.post('/auth/login', {
    email: 'admin@sigesto.com',
    password: '123456',
  });
  
  api.setToken(response.token);
  console.log('Login exitoso:', response.token);
} catch (error) {
  console.error(`Error ${error.statusCode}:`, error.message);
}
```

  "mensaje": "Validación completada.",

    "hora_preferida": "10:00",

    String endpoint,
    Map<String, String> fields,
    String filePath,
) async {
    final uri = Uri.parse('$baseUrl$endpoint');
    final request = http.MultipartRequest('POST', uri)
      ..headers.addAll(_headers);
    
    fields.forEach((key, value) {
      request.fields[key] = value;
    });
    
    request.files.add(await http.MultipartFile.fromPath('archivo', filePath));

    final response = await http.Response.fromStream(streamedResponse);

      return data;
    } else {
      throw ApiException(
        message: data['mensaje'] ?? 'Error desconocido',
        statusCode: response.statusCode,
      );
    }
}

## 8. Manejo de Errores

### 🚨 Códigos HTTP Estándar

| Código | Significado | Acción Frontend |
|--------|-------------|-----------------|
| `200 OK` | Éxito | Mostrar datos |
| `201 Created` | Recurso creado | Mostrar confirmación |
| `400 Bad Request` | Error de negocio | Mostrar mensaje de error |
| `401 Unauthorized` | Token inválido/expirado | Redirigir a login |
| `403 Forbidden` | Sin permisos | Mostrar "Acceso denegado" |
| `404 Not Found` | Recurso no existe | Mostrar "No encontrado" |
| `409 Conflict` | Conflicto (ej. horario) | Mostrar mensaje específico |
| `422 Unprocessable` | Error de validación | Mostrar errores por campo |
| `500 Internal Error` | Error del servidor | Mostrar "Error interno" |

### 📋 Formato de Errores

**Error de validación (422):**
```json
{
  "mensaje": "Error de validación.",
  "errores": {
    "email": ["El correo ya está registrado."],
    "password": ["La contraseña debe tener al menos 6 caracteres."]
  }
}
```

**Error de negocio (400/409):**
```json
{
  "mensaje": "Error al cambiar el estado.",
  "error": "No se puede cambiar el estado de FINALIZADA a PENDIENTE."
}
```

### 🟢 Java - Manejo de errores:
```java
try {
    String response = ApiClient.postJson(endpoint, data);
    // Procesar respuesta exitosa
} catch (IOException e) {
    // Error de red
    showError("Error de conexión");
}
```

### 🔵 Flutter - Manejo de errores:
```dart
try {
  final response = await ApiClient.post(endpoint, data);
  // Procesar respuesta exitosa
} on ApiException catch (e) {
  switch (e.statusCode) {
    case 401:
      // Redirigir a login
      Navigator.pushReplacementNamed(context, '/login');
      break;
    case 403:
      showError('No tienes permisos para esta acción');
      break;
    case 422:
      // Mostrar errores por campo
      final errores = e.errors as Map<String, dynamic>;
      errores.forEach((campo, mensajes) {
        print('$campo: ${mensajes.join(", ")}');
      });
      break;
    default:
      showError(e.message);
  }
}
```

### 🟡 JavaScript - Manejo de errores:
```javascript
try {

  // Procesar respuesta exitosa

  switch (error.statusCode) {
    case 401:
      window.location.href = '/login';
      break;
    case 403:
      alert('No tienes permisos para esta acción');
      break;
    case 422:
      // Mostrar errores por campo
      Object.entries(error.errors).forEach(([campo, mensajes]) => {
        mostrarErrorCampo(campo, mensajes.join(', '));
      });
      break;
    case 409:
      alert(error.message);
      break;
    default:
      alert('Error: ' + error.message);
  }
}
```

## 🎯 Flujo Completo de Ejemplo (ACTUALIZADO)

### Caso: Cliente solicita servicio → propone hora → paga → técnico trabaja

```
1. Cliente: POST /auth/login → obtener TOKEN_CLIENTE
2. Cliente: POST /solicitudes → crear solicitud CON fecha/hora preferida (PENDIENTE)
   {
     "descripcion_problema": "...",
     "direccion_servicio": "...",
     "fecha_preferida": "2026-06-20",
     "hora_preferida": "10:00"
   }
3. Admin: POST /auth/login → obtener TOKEN_ADMIN
4. Admin: GET /solicitudes/{uuid}/validar-hora-preferida → validar si hay conflicto
5. Admin: PATCH /solicitudes/{uuid}/tecnico → asignar técnico (ASIGNADA)
   - Si NO hay conflicto: usar fecha/hora preferida del cliente
   - Si HAY conflicto: coordinar otra hora
6. Técnico: POST /auth/login → obtener TOKEN_TECNICO
7. Técnico: PUT /solicitudes/{uuid} → armar cotización (COTIZADA)
8. Cliente: GET /finanzas/cuenta-bancaria → ver datos bancarios
9. Cliente: POST /finanzas/{uuid}/adelanto → pagar 50% (REVISION_PAGO)
10. Admin: PATCH /finanzas/{id_pago}/verificar → aprobar pago (APROBADA)
11. Técnico: PATCH /solicitudes/{uuid}/estado → iniciar trabajo (EN_PROCESO)
12. Técnico: POST /evidencias/subir → subir foto antes
13. Técnico: PATCH /solicitudes/{uuid}/estado → finalizar (FINALIZADA)
14. Técnico: POST /evidencias/subir → subir foto después
15. Técnico: POST /finanzas/pagar → cobrar saldo (PAGADA)
```

## 📝 Notas Importantes

### 🔒 Seguridad
- **NUNCA** guardar tokens en texto plano
- Java: Usar `SharedPreferences` con encriptación
- Flutter: Usar `flutter_secure_storage`
- JS: Usar `localStorage` o `sessionStorage`

### 🌐 URLs según entorno

| Entorno | Java | Flutter | JS |
|---------|------|---------|-----|
| Emulador Android | `http://10.0.2.2:8000/api` | `http://10.0.2.2:8000/api` | - |

| Dispositivo físico | `http://TU_IP:8000/api` | `http://TU_IP:8000/api` | - |
| Producción | `https://api.dominio.com/api` | Igual | Igual |

### ⏱️ Timeouts Recomendados
- **Login/Registro:** 10 segundos
- **Consultas normales:** 15 segundos
