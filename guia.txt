## 1. Campo de Urgencia

### 🎯 ¿Qué hace?
Permite al cliente marcar una solicitud como urgente. Las solicitudes urgentes aparecen primero en las listas del admin y técnico.

### 📡 Endpoints

#### **Crear solicitud con urgencia**
```http
POST /api/solicitudes
Authorization: Bearer {TOKEN_CLIENTE}
Content-Type: application/json

{
  "id_cliente": 1,
  "descripcion_problema": "Cortocircuito en la cocina",
  "direccion_servicio": "Av. Los Incas 456",
  "es_urgente": true,
  "fecha_preferida": "2026-07-10",
  "hora_preferida": "10:00"
}
```

**Respuesta (201):**
```json
{
  "mensaje": "Solicitud generada con éxito.",
  "ticket": "uuid-...",
  "data": {
    "uuid_solicitud": "uuid-...",
    "estado": "PENDIENTE",
    "es_urgente": true,
    "descripcion_problema": "Cortocircuito en la cocina"
  }
}
```

#### **Admin ve lista (urgentes primero)**
```http
GET /api/solicitudes
Authorization: Bearer {TOKEN_ADMIN}
```

**Respuesta (200):**
```json
{
  "data": [
    {
      "uuid_solicitud": "uuid-urgente",
      "es_urgente": true,
      "estado": "PENDIENTE",
      "descripcion_problema": "Cortocircuito..."
    },
    {
      "uuid_solicitud": "uuid-normal",
      "es_urgente": false,
      "estado": "PENDIENTE",
      "descripcion_problema": "Instalación..."
    }
  ]
}
```

#### **Admin filtra solo urgentes**
```http
GET /api/solicitudes?urgentes=1&estado=PENDIENTE
Authorization: Bearer {TOKEN_ADMIN}
```

### 🎨 Frontend (Flutter)

```dart

  final response = await api.post('/solicitudes', {

        trailing: Chip(
          label: Text(solicitud['estado']),
        ),
      ),
    );
  },
```

## 2. Notas en Evidencias

### 🎯 ¿Qué hace?
Permite al técnico agregar observaciones o notas al subir evidencias (fotos).

### 📡 Endpoints

#### **Subir evidencia con observaciones (simple)**
```http
POST /api/evidencias/subir
Authorization: Bearer {TOKEN_TECNICO}
Content-Type: multipart/form-data

uuid_solicitud: uuid-123
tipo_evidencia: FOTO_ANTES
archivo: [archivo.jpg]
observaciones: Tablero quemado, se observa daño severo en los breakers
```

**Respuesta (201):**
```json
{
  "mensaje": "Evidencia subida exitosamente.",
  "data": {
    "total_subidas": 1,
    "evidencias": [
      {
        "id_evidencia": 1,
        "tipo_evidencia": "FOTO_ANTES",
        "url_archivo": "https://res.cloudinary.com/...",
        "observaciones": "Tablero quemado, se observa daño severo en los breakers"
      }
    ]
  }
}
```

#### **Subir múltiples evidencias**
```http
POST /api/evidencias/subir
Authorization: Bearer {TOKEN_TECNICO}
Content-Type: multipart/form-data

uuid_solicitud: uuid-123
archivos[0]: [foto1.jpg]
archivos[1]: [foto2.jpg]
tipos_evidencia[0]: FOTO_ANTES
tipos_evidencia[1]: FOTO_DESPUES
observaciones: Trabajo completado exitosamente
```

#### **Ver evidencias de una solicitud**
```http
GET /api/evidencias/solicitud/{uuid}
Authorization: Bearer {TOKEN_ADMIN}
```

**Respuesta (200):**
```json
{
  "uuid_solicitud": "uuid-123",
  "total_evidencias": 2,
  "evidencias": [
    {
      "id_evidencia": 1,
      "tipo_evidencia": "FOTO_ANTES",
      "url_archivo": "https://res.cloudinary.com/...",
      "observaciones": "Tablero quemado",
      "fecha_subida": "2026-07-09 10:30:00"
    }
  ]
}
```

## 4. Liquidación 100% con Mínimo 30%

### 🎯 ¿Qué hace?
El cliente puede pagar desde el 30% hasta el 100% del total. Si paga el 100%, la cotización se marca como LIQUIDADA.

### 📡 Endpoints

#### **Cliente paga adelanto (30% - 100%)**
```http
POST /api/finanzas/{uuid}/adelanto
Authorization: Bearer {TOKEN_CLIENTE}
Content-Type: application/json

{
  "monto_pagado": 1249.50,
  "metodo_pago": "YAPE",
  "nro_operacion": "351-98765432"
}
```

**Respuesta (201):**
```json
{
  "mensaje": "Adelanto registrado exitosamente.",
  "pago": {
    "id_pago": 1,
    "monto_pagado": 1249.50,
    "estado_pago": "PENDIENTE_APROBACION",
    "tipo_pago": "ADELANTO"
  },
  "nuevo_estado_solicitud": "REVISION_PAGO",
  "monto_minimo_permitido": 374.85,
  "monto_maximo_permitido": 1249.50,
  "porcentaje_pagado": "100.00%",
  "es_liquidacion_total": true
}
```

#### **Pago menor al 30% (Error)**
```http
POST /api/finanzas/{uuid}/adelanto
Authorization: Bearer {TOKEN_CLIENTE}
Content-Type: application/json

{
  "monto_pagado": 200.00,
  "metodo_pago": "YAPE",
  "nro_operacion": "351-11122233"
}
```

**Respuesta (400):**
```json
{
  "mensaje": "Error al registrar adelanto.",
  "error": "El adelanto mínimo es el 30% del total (S/ 374.85). Monto enviado: S/ 200.00"
}
```

### 🎨 Frontend (Flutter)

```dart
// Widget para pago de adelanto
class PagoAdelantoScreen extends StatefulWidget {
  final double totalCotizacion;
  
  @override
  _PagoAdelantoScreenState createState() => _PagoAdelantoScreenState();
}

class _PagoAdelantoScreenState extends State<PagoAdelantoScreen> {
  final _controller = TextEditingController();
  double get montoMinimo => widget.totalCotizacion * 0.30;
  double get montoMaximo => widget.totalCotizacion;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Total: S/ ${widget.totalCotizacion.toStringAsFixed(2)}'),
        TextField(
          controller: _controller,
          keyboardType: TextInputType.number,
          decoration: InputDecoration(
            labelText: 'Monto a pagar',
            helperText: 'Mínimo: S/ ${montoMinimo.toStringAsFixed(2)} (30%)',
            hintText: 'Puede liquidar el 100%: S/ ${montoMaximo.toStringAsFixed(2)}',
          ),
        ),
        ElevatedButton(
          onPressed: () => _pagarAdelanto(),
          child: Text('Pagar Adelanto'),
        ),
      ],
    );
  }
  
  void _pagarAdelanto() async {
    final monto = double.parse(_controller.text);
    
    if (monto < montoMinimo) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text('El monto mínimo es S/ ${montoMinimo.toStringAsFixed(2)}')),
      );
      return;
    }
    
    final response = await api.post('/finanzas/$uuid/adelanto', {
      'monto_pagado': monto,
      'metodo_pago': 'YAPE',
      'nro_operacion': '351-12345678',
    });
    
    if (response['es_liquidacion_total']) {
      showDialog(
        context: context,
        builder: (_) => AlertDialog(
          title: Text('¡Liquidación Total!'),
          content: Text('Has pagado el 100% de la cotización.'),
        ),
      );
    }
  }
}
```

## 5. Validación Flexible de Comprobantes

### 🎯 ¿Qué hace?
El cliente debe enviar al menos uno: número de operación O comprobante (foto). Puede enviar ambos.

### 📡 Endpoints

#### **Pago con número de operación (JSON)**
```http
POST /api/finanzas/pagar
Authorization: Bearer {TOKEN_TECNICO}
Content-Type: application/json

{
  "uuid_solicitud": "uuid-123",
  "monto_pagado": 500.00,
  "metodo_pago": "TRANSFERENCIA",
  "nro_operacion": "002-99887766"
}
```

#### **Pago con comprobante (multipart/form-data)**
```http
POST /api/finanzas/pagar
Authorization: Bearer {TOKEN_TECNICO}
Content-Type: multipart/form-data

uuid_solicitud: uuid-123
monto_pagado: 500.00
metodo_pago: YAPE
comprobante: [screenshot.jpg]
```

#### **Pago sin pruebas (Error)**
```http
POST /api/finanzas/pagar
Authorization: Bearer {TOKEN_TECNICO}
Content-Type: application/json

{
  "uuid_solicitud": "uuid-123",
  "monto_pagado": 500.00,
  "metodo_pago": "YAPE"
}
```

**Respuesta (422):**
```json
{
  "mensaje": "Error de validación.",
  "errores": {
    "comprobante": ["Debe ingresar el número de operación o subir el comprobante."]
  }
}
```

### 🎨 Frontend (Flutter)

```dart
// Widget para pago con validación flexible
class PagoScreen extends StatefulWidget {
  @override
  _PagoScreenState createState() => _PagoScreenState();
}

class _PagoScreenState extends State<PagoScreen> {
  final _nroOperacionController = TextEditingController();
  File? _comprobanteFile;
  
  bool get _puedeEnviar => 
      _nroOperacionController.text.isNotEmpty || 
      _comprobanteFile != null;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        TextField(
          controller: _nroOperacionController,
          decoration: InputDecoration(
            labelText: 'Número de operación',
            helperText: 'O sube un comprobante',
          ),
        ),
        ElevatedButton(
          onPressed: () => _seleccionarImagen(),
          child: Text('Subir Comprobante'),
        ),
        if (_comprobanteFile != null)
          Image.file(_comprobanteFile!),
        ElevatedButton(
          onPressed: _puedeEnviar ? () => _registrarPago() : null,
          child: Text('Registrar Pago'),
        ),
      ],
    );
  }
  
  void _registrarPago() async {
    if (_comprobanteFile != null) {
      // Enviar como multipart
      final request = http.MultipartRequest(
        Uri.parse('${baseUrl}/api/finanzas/pagar'),
        headers: {'Authorization': 'Bearer $token'},
      );
      request.fields['uuid_solicitud'] = uuid;
      request.fields['monto_pagado'] = '500.00';
      request.fields['metodo_pago'] = 'YAPE';
      request.files.add(await http.MultipartFile.fromPath(
        'comprobante',
        _comprobanteFile!.path,
      ));
      await request.send();
    } else {
      // Enviar como JSON
      await api.post('/finanzas/pagar', {
        'uuid_solicitud': uuid,
        'monto_pagado': 500.00,
        'metodo_pago': 'TRANSFERENCIA',
        'nro_operacion': _nroOperacionController.text,
      });
    }
  }
}
```

## 6. Cloudinary para Comprobantes

### 🎯 ¿Qué hace?
Los comprobantes de pago se suben automáticamente a Cloudinary y se guarda la URL en la base de datos.

### 📡 Endpoints

#### **Subir adelanto con comprobante**
```http
POST /api/finanzas/{uuid}/adelanto
Authorization: Bearer {TOKEN_CLIENTE}
Content-Type: multipart/form-data

monto_pagado: 500.00
metodo_pago: YAPE
comprobante: [screenshot.jpg]
```

**Respuesta (201):**
```json
{
  "mensaje": "Adelanto registrado exitosamente.",
  "pago": {
    "id_pago": 1,
    "monto_pagado": 500.00,
    "url_comprobante": "https://res.cloudinary.com/dfwocmvri/image/upload/v1234567/sigesto/pagos/abc123.jpg"
  }
}
```

### 🎨 Frontend (Flutter)

```dart
// El frontend solo necesita subir el archivo, Cloudinary se maneja en el backend
void _subirAdelantoConComprobante() async {
  final request = http.MultipartRequest(
    Uri.parse('${baseUrl}/api/finanzas/$uuid/adelanto'),
    headers: {'Authorization': 'Bearer $token'},
  );
  
  request.fields['monto_pagado'] = '500.00';
  request.fields['metodo_pago'] = 'YAPE';
  request.files.add(await http.MultipartFile.fromPath(
    'comprobante',
    selectedImage.path,
  ));
  
  final response = await request.send();
  final data = jsonDecode(response.stream.toString());
  
  // La URL de Cloudinary viene en la respuesta
  print('URL del comprobante: ${data['pago']['url_comprobante']}');
}
```

```dart
class CotizacionScreen extends StatefulWidget {

  _CotizacionScreenState createState() => _CotizacionScreenState();

class _CotizacionScreenState extends State<CotizacionScreen> {
  List<dynamic> _tiposTrabajo = [];
  List<dynamic> _itemsSugeridos = [];
  int? _tipoTrabajoSeleccionado;
  
  @override
  void initState() {
    super.initState();
    _cargarTiposTrabajo();
  }
  
  Future<void> _cargarTiposTrabajo() async {
    final response = await api.get('/catalogo/tipos-trabajo');
    setState(() {
      _tiposTrabajo = response['data']['tipos_trabajo'];
    });
  }
  
  Future<void> _cargarSugerencias(int idTipoTrabajo) async {
    final response = await api.get('/catalogo/tipos-trabajo/$idTipoTrabajo/materiales-sugeridos');
    setState(() {
      _itemsSugeridos = response['data']['items_sugeridos'];
    });
  }
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        DropdownButton<int>(
          hint: Text('Seleccione tipo de trabajo'),
          value: _tipoTrabajoSeleccionado,
          items: _tiposTrabajo.map((tipo) {
            return DropdownMenuItem(
              value: tipo['id'],
              child: Text(tipo['nombre']),
            );
          }).toList(),
          onChanged: (id) {
            setState(() => _tipoTrabajoSeleccionado = id);
            _cargarSugerencias(id!);
          },
        ),
        
        if (_itemsSugeridos.isNotEmpty)
          ListView.builder(
            shrinkWrap: true,
            itemCount: _itemsSugeridos.length,
            itemBuilder: (context, index) {
              final item = _itemsSugeridos[index];
              return Card(
                child: ListTile(
                  title: Text(item['nombre']),
                  subtitle: Text(
                    'Cantidad: ${item['cantidad_sugerida']} ${item['unidad_medida']}\n'
                    'Precio: S/ ${item['precio_ref']}'
                  ),
                  trailing: Text('S/ ${item['subtotal_sugerido']}'),
                ),
              );
            },
          ),
        
        ElevatedButton(
          onPressed: () => _enviarCotizacion(),
          child: Text('Enviar Cotización'),
        ),
      ],
    );
  }
  
  void _enviarCotizacion() async {
    final items = _itemsSugeridos.map((item) => {
      'id_item': item['id_item'],
      'cantidad': item['cantidad_sugerida'],
      'precio_aplicado': item['precio_ref'],
    }).toList();
    
    await api.put('/solicitudes/$uuid', {
      'items': items,
    });
  }
}
```

```dart
      child: Padding(
        padding: EdgeInsets.all(16),

            Text(title, style: TextStyle(fontSize: 14)),
            SizedBox(height: 8),
            Text(value, style: TextStyle(fontSize: 24, fontWeight: FontWeight.bold)),
          ],
        ),
      ),
    );
  }
}
```

## 📊 Resumen de Endpoints por Rol

### **Cliente**
- `POST /api/solicitudes` - Crear solicitud (con `es_urgente`, `materiales_cliente`)
- `GET /api/solicitudes/mis-solicitudes` - Ver sus solicitudes
- `POST /api/finanzas/{uuid}/adelanto` - Pagar adelanto (30%-100%)
- `GET /api/finanzas/mis-pagos` - Ver sus pagos

### **Técnico**
- `GET /api/solicitudes/asignadas` - Ver hoja de ruta (urgentes primero)
- `POST /api/evidencias/subir` - Subir evidencias (con `observaciones`)
- `GET /api/catalogo/tipos-trabajo` - Ver tipos de trabajo
- `GET /api/catalogo/tipos-trabajo/{id}/materiales-sugeridos` - Ver sugerencias
- `PUT /api/solicitudes/{uuid}` - Enviar cotización

### **Admin**
- `GET /api/solicitudes` - Ver todas (urgentes primero, filtros)
- `GET /api/solicitudes?urgentes=1&estado=PENDIENTE` - Filtrar
- `PATCH /api/finanzas/{id_pago}/verificar` - Aprobar pago
- `GET /api/reportes/dashboard` - Dashboard
- `GET /api/reportes/ingresos` - Reporte de ingresos
- `GET /api/reportes/cotizaciones` - Reporte de cotizaciones
- `GET /api/reportes/pagos-pendientes` - Pagos pendientes

## 🎯 Casos de Uso Comunes

### **Flujo Completo de Solicitud Urgente**

1. **Cliente** crea solicitud con `es_urgente: true`
2. **Admin** ve la solicitud primero en la lista
3. **Admin** asigna técnico
4. **Técnico** recibe solicitud urgente primero en su hoja de ruta
5. **Técnico** sube evidencias con observaciones
6. **Técnico** envía cotización
7. **Cliente** paga 100% (liquidación)
8. **Admin** verifica pago → Cotización pasa a LIQUIDADA

### **Flujo de Cotización con Sugerencias**

1. **Técnico** abre formulario de cotización
2. **Técnico** selecciona "Instalación de tablero"
3. **Sistema** sugiere: Tablero (1x), Cable (5x), Mano de obra (1x)
4. **Técnico** ajusta cantidades si es necesario
5. **Técnico** envía cotización
